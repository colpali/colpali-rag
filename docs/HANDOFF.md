# Handoff — start here

Front door for the next engineer or agent working on **colpali-rag**, a visual
document RAG: index PDF pages as images (no OCR), retrieve with ColPali late
interaction, show a heatmap of where the model looked, and optionally answer
questions with a vendor-neutral vision model.

> One-line mental model: **PDF pages → ColPali patch multivectors → MaxSim retrieval
> → per-token heatmap**, with an optional **retrieve-then-read** answer step.

## Repo map

```
src/colpali_rag/
  config.py     env-driven settings (model, device, store, qdrant, optional answer model)
  pdf.py        PDF -> page images + text (pypdfium2; no OCR)
  embedder.py   ColPali wrapper: retrieval (MaxSim) + similarity-map interpretability
  heatmap.py    similarity grid -> aligned inferno overlay (NumPy + Pillow)
  store.py      MemoryStore (brute-force MaxSim) + QdrantStore (native MAX_SIM)
  generator.py  optional, vendor-neutral answer model (any OpenAI-compatible endpoint)
  engine.py     build_index / open_index
  app.py        FastAPI service + self-contained web UI (search · heatmap · ask)
  cli.py        colpali-rag index | query | serve | info
tests/          model-free tests (corpus generated by conftest.py)
docker-compose.yml   optional Qdrant
```

## Quickstart

```bash
python3 -m venv .venv && source .venv/bin/activate
pip install -e '.[rag,api,dev]'

colpali-rag index ./pdfs        # any folder of PDFs
colpali-rag serve               # http://127.0.0.1:8000
pytest -q
```

## Gotchas / non-obvious

- **The heatmap runs on a non-split single-image pass.** ColPali models tile large
  images; interpretability needs one clean patch grid, so `embedder.similarity_maps`
  temporarily disables image splitting. See [COLPALI.md](COLPALI.md).
- **Filter out padding tokens.** ColPali pads queries with special tokens; per-token
  heatmaps must use only content tokens (handled in `similarity_maps`).
- **Model forwards aren't re-entrant.** The app serializes them behind a lock.
- **Mac MPS:** pin `torch==2.5.1` if `COLPALI_DEVICE=mps`; `cpu` avoids it.
- **Config over code:** models, device, store, Qdrant URL, and the optional answer
  model all come from env / `.env` (see `.env.example`). `src/` hard-codes nothing.
- **Answers are optional and vendor-neutral.** Set `VLM_BASE_URL` to any
  OpenAI-compatible vision endpoint; leave it empty and search/heatmaps still work.
